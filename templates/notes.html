Lesson 7 - Object-Oriented Programming

Object-Oriented Programming

Object-Oriented Programming or OPP for short focuses on using
Objects rather then Actions. Its designed to make coding easyer
for programmers. OOP languages used are Java, Python, C++,
Visual Basic .NET and Ruby. So what is an Object in OOP?
Well think of each Object as a small program able to preform
a couple of operations. By by its self it might not do that much
but combine matiple objects together and you are able to create very
complex programs. OOP makes use of one of the key concepts you keep
hearing Remember "Avoinding Repetition" also helps programmers
save time.

Major conepts in OOP

  1. Inheritance
  2. Method Over-Riding

Inheritance - is when an object or class is based on another object (prototypal inheritance)
or class (class-based inheritance), using the same implementation
(inheriting from an object or class) specifying implementation
to maintain the same behavior.

Method Over-Riding - is a language feature that allows a subclass or child class
to provide a specific implementation of a method that is already provided by one of
its superclasses or parent classes.

OOP Vocabulary

  1. Object - Objects contain two things data and procedures.
    What an Object does is take the data and runs
    the procedures on the data prodecing an output.

  2. Instance - is a specific realization of any object. An object may
    be varied in a number of ways. Each realized variation of that
    object is an instance.

  3. Class - is the blueprint or template used for creating objects.

  4. Method - is a programmed procedure that is defined as part of a
    class and included in any object of that class.

  5. Module - is how we would define a class. Basically giving it a name.
    Then you use an import statement you are
    importing a Module.

  6. Library - is where you would store your
    objects. So that you are able to re-use them.
